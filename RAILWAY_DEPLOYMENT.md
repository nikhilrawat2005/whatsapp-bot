# 🌐 Railway Deployment Guide

This guide provides step-by-step instructions to host your AI-powered Hospital Receptionist & Appointment Booking application on **Railway**.

---

## 🏗️ 1. Project Prerequisites
The repository is fully configured for a production deploy:
- **`Procfile`**: Specifies `web: gunicorn app:app` for production serving.
- **`requirements.txt`**: Declares all dependencies, including `openpyxl` for Excel exports and `gunicorn`.
- **Database Routing**: Automatically switches to the PostgreSQL driver if the `DATABASE_URL` environment variable is detected.

---

## 🚀 2. Deploying on Railway

### Step 1: Connect your Repository
1. Log in to [Railway](https://railway.app/).
2. Click **New Project** → **Deploy from GitHub repo**.
3. Select your repository `whatsapp-bot` (or the renamed repo).

### Step 2: Add a PostgreSQL Database
1. Inside your Railway project canvas, click **+ Add** or **New**.
2. Select **Database** → **Add PostgreSQL**.
3. Railway will provision a PostgreSQL instance and automatically inject the connection string into your application via the `DATABASE_URL` environment variable.

### Step 3: Configure Environment Variables
In your web service block, navigate to the **Variables** tab and set the following required variables:

| Variable | Recommended Value | Description |
|---|---|---|
| `FLASK_ENV` | `production` | Sets the Flask environment mode |
| `SECRET_KEY` | `your-long-random-string-here` | Secret key for securing sessions |
| `PORT` | `5000` | Port to bind (Railway automatically routes this) |
| `HOSPITAL_NAME` | `Your Hospital Name` | Display name of the hospital across the platform |
| `HOSPITAL_PHONE` | `+1 (234) 567-8900` | Contact phone number |
| `HOSPITAL_EMAIL` | `contact@yourhospital.com` | Contact email address |
| `HOSPITAL_ADDRESS` | `123 Health Ave, Clinic District` | Street address of the hospital |

---

## 🛠️ 3. How the DB Migration Works
On startup, the application factory (`app/__init__.py`):
1. Detects PostgreSQL configuration through `DATABASE_URL`.
2. Connects to PostgreSQL and provisions all tables (`patients`, `available_slots`, `web_chat_sessions`, `chat_messages`, `hospital_settings`, `doctors`).
3. Automatically seeds the database with the default settings and lists of doctors.
4. Dynamically generates available slots for the next 30 days.

---

## 🧪 4. Post-Deployment Verification
1. Open the public domain URL generated by Railway (e.g. `https://your-app.up.railway.app`).
2. Interact with the **AI Receptionist** floating widget at the bottom-right:
   - Click **Book Appointment** to verify slots are loading correctly.
   - Complete the booking flow step-by-step.
3. Access the Admin Dashboard at `/admin` (e.g. `https://your-app.up.railway.app/admin`):
   - Confirm your test booking appears instantly.
   - Check the **Analytics** page for visual summaries.
   - Edit hospital working hours or doctor assignments in the **Settings** view.
