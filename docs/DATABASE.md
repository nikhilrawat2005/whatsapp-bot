# 🗄️ Database Architecture & Fallback Engine Guide

This document describes the design of the database engine in the Hospital Appointment Booking WhatsApp Bot, details how to switch engines between local development and production, and lists the schema definitions.

---

## 1. Multi-Engine Architecture
The database layer uses **SQLAlchemy ORM** and is designed to run seamlessly in different environments:
* **Development (SQLite)**: If the environment variable `DATABASE_URL` is omitted, the app automatically initializes and uses a local SQLite database file at `instance/hospital.db`. This requires zero setup for developer onboarding.
* **Production (PostgreSQL)**: When deployed in production (e.g., Render), setting `DATABASE_URL` prompts the system to connect to the production PostgreSQL server. The configuration dynamically replaces `postgres://` prefixes with `postgresql://` to accommodate standard connection strings generated by cloud platforms.

---

## 2. Table Schemas

### 2.1 `patients`
Stores registered patient data and booking records:
* `id` (Integer, Primary Key): Unique auto-incrementing ID.
* `name` (String, Nullable=False): Patient's full name.
* `age` (Integer, Nullable=False): Patient's age.
* `gender` (String, Nullable=False): Patient's gender (Male, Female, Other).
* `problem` (Text): Details of the symptom or consultation reason.
* `doctor` (String, Nullable=False): Assigned physician specialization.
* `booking_date` (String, Nullable=False): Appointment date (YYYY-MM-DD).
* `booking_time` (String, Nullable=False): Time slot (HH:MM).
* `phone_number` (String, Nullable=False): Sender's WhatsApp number.
* `booking_status` (String, Nullable=False, Default='Scheduled'): Booking lifecycle state (`Scheduled`, `Completed`, `Cancelled`).
* `created_at` (DateTime, Default=UTC Now): Timestamp of registration.

### 2.2 `available_slots`
Holds pre-allocated doctor consultation slots:
* `id` (Integer, Primary Key): Unique ID.
* `doctor` (String, Nullable=False): Specialization of the doctor.
* `date` (String, Nullable=False): Slot date (YYYY-MM-DD).
* `time` (String, Nullable=False): Slot time (HH:MM).
* `is_booked` (Integer, Nullable=False, Default=0): Boolean flag (0 for free, 1 for booked).
* `patient_id` (Integer, Foreign Key ➔ `patients.id`): References the patient holding the slot.

*Unique Constraint*: Unique composite key `(doctor, date, time)` prevents duplicate slots.

### 2.3 `bot_states`
Maintains conversational state machines for interactive WhatsApp sessions:
* `phone_number` (String, Primary Key): WhatsApp number of the session owner.
* `state` (String, Default='WELCOME'): Conversation step (e.g. `AWAITING_NAME`, `AWAITING_AGE`, etc.).
* `temp_name`, `temp_age`, `temp_gender`, `temp_problem`, `temp_doctor`, `temp_date`, `temp_slot` (Temp fields): Holds conversational variables until the patient confirms the final booking.
* `updated_at` (DateTime): Tracks last activity time.
